# 学生请销假系统 — 方案设计

> 小学期课设。Spring Boot 3 + Spring AI + Vue 3 + 微信小程序(uni-app)，Apple HIG 风格 UI。
> 参考真实校园请销假系统（SNNU）做多级审批与辅助功能。

## 1. 技术选型

| 层 | 技术 | 说明 |
|---|---|---|
| 后端 | Spring Boot 3.4 (Java 17) | REST，端口 8080，上下文 `/api` |
| ORM | MyBatis-Plus 3.5.9 | 实体/条件构造/分页 |
| AI | Spring AI 1.0（OpenAI + Anthropic 双供应商） | **优先 OpenAI、回退 Anthropic**，model/base-url 环境变量可自定义 |
| 认证 | JWT (jjwt 0.12) + BCrypt | 无状态令牌 + 密码哈希 |
| 数据库 | MySQL 8 / InnoDB，库 `leave_sys`（9 表） | 见 [数据库建模.md](数据库建模.md) |
| Web 前端 | Vue 3 + Vite + Vue Router + Pinia + Axios + ECharts | dev 5173，`/api` 代理 8080 |
| 移动端 | 微信小程序 uni-app(Vue3+Vite) | 编译 mp-weixin + h5；微信一键登录 |
| 报表 | EasyExcel / POI（请假数据导出 xlsx） | 辅导员/副书记/管理员导出台账 |
| UI | 自建 Apple HIG 设计体系（见 §6） | 无第三方 UI 库、无 emoji |

- AI 无 Key 时优雅降级（code=5001，前端提示但不阻断人工流程）；出站带 5s/60s 超时。

## 2. 角色与多级审批业务流程

四角色：**STUDENT 学生 / TEACHER 辅导员 / LEADER 副书记 / ADMIN 管理员**。
审批为**多级**：普通请假辅导员一级审批即可；**超过该请假类型上限天数（`leave_type.max_days`）的长假，辅导员通过后需再经副书记二级审批**。

```
学生提交请假单 ──► PENDING 待辅导员审批
   │撤回(仅PENDING)        │辅导员审批
   ▼                      ├─ 驳回 ─────────────► REJECTED（终态）
 REVOKED（终态）           └─ 通过 ─► 天数 > 类型上限 ?
                                       ├─是─► LEADER_PENDING 待副书记审批
                                       │        │副书记审批
                                       │        ├─ 驳回 ─► REJECTED
                                       │        └─ 通过 ─► APPROVED
                                       └─否─────────────► APPROVED 请假中
                                                             │学生返校申请销假
                                                             ▼
                                                       CANCEL_PENDING 销假待确认
                                                             │辅导员确认
                                                             ▼
                                                       COMPLETED 已完成（终态）
```

- 请假类型（字典表 `leave_type`）：SICK 病假(≤15天) / PERSONAL 事假(≤7) / EMERGENCY 急事(≤3) / OTHER 其他(≤5)；超上限触发副书记审批。
- 学生绑定辅导员（`sys_user.teacher_id`），辅导员只审名下学生；副书记审全校长假。
- 所有审批/销假动作写入 `approval_record` 审计表（操作人/动作/意见/时间），详情页时间线展示。
- 六→**七状态**：PENDING / **LEADER_PENDING** / APPROVED / REJECTED / REVOKED / CANCEL_PENDING / COMPLETED。

## 3. Spring AI 三能力

| 接口 | 功能 | 实现 |
|---|---|---|
| AI 智能填单 | 学生一句话（"下周一到周三回家参加婚礼"）→ 结构化请假草稿 | ChatClient + 提示词约束输出 JSON + 当天日期注入 + 后端解析校验 |
| AI 审批建议 | 辅导员点"AI 建议" → 结合该生近半年请假统计给风险等级+建议 | 后端拼上下文 → ChatClient |
| AI 制度问答 | 请假制度问答浮窗 | system 提示词内置制度（可由 `sys_config` 后台配置），对话留痕 `ai_chat_log` |

供应商 auto=优先 OpenAI、回退 Anthropic；均无 Key → 5001 降级不阻断人工。

## 4. 数据库设计（9 表 · 2 子系统）

- **核心业务域(5)**：sys_user 用户、leave_request 请假单、approval_record 审批记录、leave_attachment 附件、leave_type 类型字典
- **系统支撑域(4)**：sys_notification 通知、ai_chat_log AI记录、sys_operation_log 操作日志、sys_config 配置

概念/逻辑/物理三级模型、E-R 图、范式分析见 [数据库建模.md](数据库建模.md)。DDL 见 [schema.sql](schema.sql)。每张表均由后端真实使用。

## 5. 模块划分（后端包结构）

```
com.school.leave
├─ config/        JWT鉴权拦截器、@RequireRole、MyBatis-Plus、CORS、操作日志拦截器、AI出站超时、sys(配置)
├─ common/        Result 统一响应、BizException、UserContext、枚举(角色/类型/状态)
├─ auth/          账号密码登录 + 微信一键登录/绑定、JWT
├─ user/          用户与角色
├─ leave/         请假单：提交/撤回/销假/附件；多级审批状态机
├─ approval/      辅导员/副书记审批、销假确认、请假次数排名、数据导出
├─ dict/          请假类型字典
├─ notify/        消息通知
├─ oplog/         操作日志
├─ admin/         管理端用户CRUD/统计/配置
└─ ai/            AiController + AiService(双供应商 ChatClient) + ai_chat_log
```

## 6. 前端 Apple HIG 设计规范

CSS 变量令牌落地，禁硬编码颜色、禁 emoji 图标（内联 SVG，`stroke=currentColor`）：

```css
:root{
  --accent:#0071e3; --bg:#f5f5f7; --surface:rgba(255,255,255,.72);
  --text-1:#1d1d1f; --text-2:#6e6e73; --separator:rgba(0,0,0,.08);
  --radius-lg:18px; --radius-md:12px; --radius-pill:980px; --shadow-card:0 4px 24px rgba(0,0,0,.06);
}
```
- 毛玻璃侧边栏/弹窗（小程序降级 rgba 高透明）；胶囊主按钮；iOS 状态徽章 pill（待审批橙/请假中绿/驳回红/完成灰/待副书记审批蓝）；分段控件；34px 大标题；150–250ms 动效。
