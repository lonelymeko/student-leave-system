# 学生请销假系统 — 接口文档 v1.0

Base URL: `http://localhost:8080/api`　|　认证：`Authorization: Bearer <JWT>`（除登录外全部需要）

统一响应：
```json
{ "code": 0, "msg": "ok", "data": {} }
```
| code | 含义 |
|---|---|
| 0 | 成功 |
| 401 | 未登录/Token失效 |
| 403 | 无权限（角色不符/越权访问他人数据） |
| 4001 | 参数校验失败 |
| 4004 | 资源不存在 |
| 4009 | 状态不允许该操作（如审批已终态单据） |
| 4010 | 微信未绑定账号（data 返回 bindTicket，去绑定页） |
| 5001 | AI 服务不可用（未配 Key/调用失败） |
| 5002 | 微信登录未启用或微信侧调用失败 |
| 500 | 服务器错误 |

分页统一返回：`{ "records": [...], "total": 57, "current": 1, "size": 10 }`

## 枚举

- 角色 role：`STUDENT` / `TEACHER` / `ADMIN`
- 请假类型 type：`SICK`病假 / `PERSONAL`事假 / `EMERGENCY`急事 / `OTHER`其他
- 状态 status：`PENDING`待审批 / `APPROVED`请假中 / `REJECTED`已驳回 / `REVOKED`已撤回 / `CANCEL_PENDING`销假待确认 / `COMPLETED`已完成

## 1. 认证

### POST /auth/login
入参 `{ "username": "student1", "password": "123456" }`
出参 data：
```json
{ "token": "eyJ...", "user": { "id": 3, "username": "student1", "realName": "张三",
  "role": "STUDENT", "studentNo": "20230001", "className": "软件2101", "teacherName": "李辅导" } }
```

### GET /auth/me　→ 同上 user 对象

### GET /auth/wx-enabled　微信登录是否可用（免登录）
出参 `{ "enabled": true }`（后端未配 `WX_APPID`/`WX_SECRET` 时 false，小程序隐藏微信按钮）

### POST /auth/wx-login　微信一键登录（免登录，小程序端）
入参 `{ "code": "wx.login() 拿到的临时code" }`
- 已绑定：`code=0`，data 同 /auth/login（token + user）
- 未绑定：`code=4010`，data `{ "bindTicket": "10分钟有效的绑定票据" }` → 跳绑定页
- 未启用/微信侧错误：`code=5002`

### POST /auth/wx-bind　微信绑定并登录（免登录）
入参 `{ "ticket": "<bindTicket>", "username": "student1", "password": "123456" }`
校验账号密码后将该微信 openid 绑定到账号并直接返回 token+user。
账号已绑其他微信 / 微信已绑其他账号 → `4009`；票据无效过期 → `4001`。

## 2. 学生端（role=STUDENT）

### POST /leave　提交请假
```json
{ "type": "PERSONAL", "startTime": "2026-07-13 08:00:00", "endTime": "2026-07-15 18:00:00",
  "reason": "回家参加表哥婚礼", "destination": "乌鲁木齐", "contactPhone": "13800000000" }
```
校验：endTime>startTime；startTime 不得早于今天 0 点；reason 5~200字。`days` 后端按自然天数计算。出参：完整 LeaveVO。

### GET /leave/my?status=&page=1&size=10　我的请假分页
LeaveVO 字段：`id,type,typeText,startTime,endTime,days,reason,destination,contactPhone,status,statusText,approverName,approveComment,approveTime,cancelApplyTime,cancelNote,completeTime,createTime,studentName,studentNo,className`

### GET /leave/{id}　详情（学生只能看自己的，否则 403）
出参：LeaveVO + `records`: 审批时间线 `[{operatorName, action, actionText, comment, createTime}]`
action：`SUBMIT/APPROVE/REJECT/REVOKE/CANCEL_APPLY/CANCEL_CONFIRM`

### PUT /leave/{id}/revoke　撤回（仅 PENDING 可撤，否则 4009）

### POST /leave/{id}/cancel-apply　申请销假（仅 APPROVED）
`{ "note": "已于15日晚返校" }`　→ 状态变 CANCEL_PENDING

## 3. 辅导员端（role=TEACHER，只看得到 teacher_id=自己 的学生）

### GET /approval/pending?page=&size=　待办：status ∈ (PENDING, CANCEL_PENDING)，PENDING 在前
### GET /approval/history?page=&size=　我处理过的
### POST /approval/{leaveId}　审批（仅 PENDING）
`{ "action": "APPROVE" | "REJECT", "comment": "同意，注意安全" }`（REJECT 时 comment 必填）
### POST /approval/{leaveId}/cancel-confirm　确认销假（仅 CANCEL_PENDING）→ COMPLETED

## 4. 管理端（role=ADMIN）

### GET /admin/users?role=&keyword=&page=&size=　用户分页（keyword 模糊 username/realName/studentNo）
### POST /admin/users　新增
`{ "username","password","realName","role","studentNo?","className?","phone?","teacherId?" }`（STUDENT 必填 teacherId）
### PUT /admin/users/{id}　编辑（不含密码）
### PUT /admin/users/{id}/password　重置密码 `{ "password": "..." }`
### DELETE /admin/users/{id}　逻辑删除（status=0，不可登录）
### GET /admin/teachers　辅导员下拉 `[{id, realName}]`
### GET /admin/leaves?status=&keyword=&page=&size=　全部请假单
### GET /admin/stats/overview　统计

{ "totalCount": 57, "pendingCount": 4, "approvedCount": 8, "completedCount": 40,
  "typeDist": [ {"type":"SICK","name":"病假","count":20}, ... ],

## 5. AI 接口（Spring AI · 双供应商：优先 OpenAI、回退 Anthropic，model/endpoint 可环境变量自定义）

> 未配置任何供应商 Key（`OPENAI_API_KEY`/`ANTHROPIC_API_KEY`）或调用异常 → `code=5001, msg="AI服务暂不可用"`，前端提示但不阻断人工流程。

### POST /ai/draft　智能填单（STUDENT）
入参 `{ "text": "我下周一到周三回家参加表哥的婚礼，去乌鲁木齐" }`
出参 data（后端注入当天日期到提示词，要求模型只输出 JSON 并解析校验）：
```json
{ "type": "PERSONAL", "startTime": "2026-07-13 08:00:00", "endTime": "2026-07-15 18:00:00",
  "reason": "回家参加表哥婚礼", "destination": "乌鲁木齐" }
```

### POST /ai/approval-advice　审批建议（TEACHER）
入参 `{ "leaveId": 12 }`。后端拼上下文：本单信息 + 该生近半年请假次数/总天数/驳回次数。
出参 `{ "riskLevel": "LOW|MEDIUM|HIGH", "advice": "该生本学期第3次事假……建议通过，提醒补交课堂笔记。" }`

### POST /ai/chat　制度问答（登录即可）
入参 `{ "message": "病假超过3天需要什么材料？" }`　出参 `{ "reply": "……" }`
system 提示词优先读取 `sys_config` 表 `leave.policy`（见 §7.2），读不到回退内置常量。成功产出回复后 best-effort 写入 `ai_chat_log`（记录失败不影响返回）。

## 7. 增量子系统（迭代二 · 全部为“加法”，不改动既有接口语义）

> 统一响应 `{code,msg,data}`。除标注外均需登录（Bearer Token）。

### 7.1 请假类型字典

#### GET /leave-types　类型列表（登录即可）
返回 `enabled=1` 且按 `sort` 升序的类型；学生提交请假时 `type` 必须命中字典中启用的 `typeCode`，否则 `4001`。
```json
{ "code":0, "msg":"ok", "data":[
  {"id":1,"typeCode":"SICK","typeName":"病假","maxDays":15,"needProof":1,"sort":1,"enabled":1},
  {"id":2,"typeCode":"PERSONAL","typeName":"事假","maxDays":7,"needProof":0,"sort":2,"enabled":1}
]}
```

### 7.2 系统配置（role=ADMIN）

#### GET /admin/configs　配置列表
```json
{ "code":0,"msg":"ok","data":[
  {"id":1,"configKey":"ai.provider","configValue":"auto","description":"AI供应商","updateTime":"..."},
  {"id":3,"configKey":"leave.policy","configValue":"病假超过3天……","description":"请假制度"}
]}
```

#### PUT /admin/configs/{key}　更新配置值
入参 `{ "value": "新的配置值" }`。key 不存在则新建。出参 `data=null`。
`leave.policy` 会被 AI 制度问答用作 system 提示词。

### 7.3 消息通知（当前登录用户）

审批（APPROVE/REJECT）与销假确认成功后，系统 best-effort 给该请假单学生推送一条通知（`bizType=APPROVAL_RESULT`，`bizId=leaveId`），失败不影响审批主流程。

#### GET /notifications?page=1&size=10　我的通知分页
```json
{ "code":0,"msg":"ok","data":{"records":[
  {"id":1,"userId":4,"title":"请假申请已通过","content":"您的病假申请已通过，意见：……",
   "bizType":"APPROVAL_RESULT","bizId":15,"isRead":0,"createTime":"..."}
], "total":3,"current":1,"size":10 }}
```

#### GET /notifications/unread-count　未读数
出参 `{ "count": 2 }`

#### PUT /notifications/{id}/read　标记已读（仅本人，否则 403；不存在 4004）
出参 `data=null`

### 7.4 请假附件

#### POST /leave/{id}/attachment　上传附件（仅该单学生本人，multipart）
表单字段 `file`（≤10MB）。存到服务端 `app.upload-dir`（默认 `./uploads`），插入元数据。非本人 `403`。
```json
{ "code":0,"msg":"ok","data":{
  "id":1,"leaveId":15,"fileName":"proof.txt",
  "fileUrl":"/uploads/leave15_1784018735923_ffd67e51.txt",
  "fileSize":32,"fileType":"text/plain","uploadTime":"..." }}
```
文件可直接静态访问：`GET /api/uploads/{存储文件名}`（免鉴权）。

#### GET /leave/{id}/attachments　附件列表（受详情越权规则约束）
出参 `data` 为附件数组；请假详情 `GET /leave/{id}` 的 `data` 也新增 `attachments` 字段。

### 7.5 操作日志（内部，无接口）
所有写操作（POST/PUT/DELETE，`/auth/login` 除外）经拦截器 best-effort 落 `sys_operation_log`（`user_id`(可空)/`operation`/`method`(HTTP方法+URI)/`ip`），失败不影响请求。

## 6. 数据库（docs/schema.sql 为准）

```sql
CREATE TABLE sys_user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE NOT NULL,
  password VARCHAR(100) NOT NULL,          -- BCrypt
  real_name VARCHAR(50) NOT NULL,
  role VARCHAR(20) NOT NULL,               -- STUDENT/TEACHER/ADMIN
  student_no VARCHAR(30), class_name VARCHAR(50), phone VARCHAR(20),
  teacher_id BIGINT,                       -- 学生的辅导员
  status TINYINT DEFAULT 1,                -- 1正常 0禁用
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE TABLE leave_request (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  student_id BIGINT NOT NULL,
  type VARCHAR(20) NOT NULL, start_time DATETIME NOT NULL, end_time DATETIME NOT NULL,
  days DECIMAL(4,1) NOT NULL, reason VARCHAR(200) NOT NULL,
  destination VARCHAR(100), contact_phone VARCHAR(20),
  status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  approver_id BIGINT, approve_comment VARCHAR(200), approve_time DATETIME,
  cancel_apply_time DATETIME, cancel_note VARCHAR(200), complete_time DATETIME,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_student(student_id), KEY idx_status(status)
);
CREATE TABLE approval_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  leave_id BIGINT NOT NULL, operator_id BIGINT NOT NULL,
  action VARCHAR(30) NOT NULL, comment VARCHAR(200),
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP, KEY idx_leave(leave_id)
);
```

种子数据（密码全部 `123456` 的 BCrypt，admin 为 `admin123`）：
- admin（ADMIN，系统管理员）
- teacher1 李辅导 / teacher2 王辅导（TEACHER）
- student1 张三 / student2 李四（teacher1 名下），student3 王五（teacher2 名下），班级"软件2101/2102"
- 预置 8~10 条覆盖各状态的请假单 + 对应审批记录（供统计页和演示）
