# 学生请销假系统 — 方案设计

> 小学期课设。SpringBoot 3 + Spring AI + Vue 3，Apple HIG 风格 UI。

## 1. 技术选型

| 层 | 技术 | 版本 |
|---|---|---|
| 后端 | Spring Boot | 3.4.x (Java 17) |
| ORM | MyBatis-Plus (spring-boot3-starter) | 3.5.9+ |
| AI | Spring AI (spring-ai-starter-model-anthropic) | 1.0.x |
| 认证 | JWT (jjwt 0.12) + BCrypt | |
| 数据库 | MySQL 8 (本机)，库名 `leave_sys` | |
| 前端 | Vue 3 + Vite + Vue Router + Pinia + Axios | |
| 图表 | ECharts (仅管理端统计页) | |
| UI | 自建 Apple HIG 设计体系（见 §5） | |

- 后端端口 **8080**，上下文 `/api`；前端 dev **5173**，`/api` 代理到 8080。
- AI 模型：Anthropic `claude-opus-4-8`，Key 走环境变量 `ANTHROPIC_API_KEY`；**无 Key 时 AI 接口优雅降级**（返回 code=5001，前端隐藏/提示，不影响主流程）。

## 2. 角色与核心业务流程

三角色：**STUDENT 学生 / TEACHER 辅导员 / ADMIN 管理员**。

```
学生提交请假单 ──► PENDING 待审批
      │撤回                │辅导员审批
      ▼                    ├─ 驳回 ──► REJECTED（终态）
   REVOKED（终态）          └─ 通过 ──► APPROVED 请假中
                                        │学生返校，申请销假
                                        ▼
                                  CANCEL_PENDING 销假待确认
                                        │辅导员确认
                                        ▼
                                  COMPLETED 已完成（终态）
```

- 请假类型：SICK 病假 / PERSONAL 事假 / EMERGENCY 急事 / OTHER 其他。
- 每个学生绑定一名辅导员（`sys_user.teacher_id`），审批只到自己名下学生。
- 所有审批/销假动作写入 `approval_record` 审计表（操作人/动作/意见/时间）。

## 3. Spring AI 三个能力（本课设亮点）

| 接口 | 功能 | 实现 |
|---|---|---|
| AI 智能填单 | 学生输入自然语言（"我下周一到周三回家参加表哥婚礼"）→ 返回结构化请假单草稿(类型/起止/事由) | ChatClient + 提示词约束输出 JSON，后端解析校验后返回 |
| AI 审批建议 | 辅导员点"AI 建议" → 结合该生历史请假次数/本次时长/事由，生成审批建议与风险等级 | 后端拼装上下文 → ChatClient |
| AI 助手问答 | 关于请假制度的简单问答 | ChatClient + system 提示词内置院系请假制度 |

系统提示词中固定"只输出 JSON"并用 BeanOutputConverter/手动解析；当天日期由后端注入提示词（模型不知道今天几号）。

## 4. 模块划分（后端包结构）

```
com.school.leave
├─ config/        SecurityConfig(JWT过滤器)、MybatisPlusConfig、CorsConfig、AiConfig
├─ common/        Result<T>统一响应、全局异常、常量/枚举
├─ auth/          登录、JWT签发/解析、@CurrentUser
├─ user/          管理员用户CRUD
├─ leave/         请假单：学生提交/撤回/销假申请；辅导员审批/销假确认；管理端查询
├─ stats/         管理端统计
└─ ai/            AiController + AiService(Spring AI ChatClient)
```

## 5. 前端 Apple HIG 设计规范（frontend agent 必须遵守）

参考 Apple [Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines) 与 [Puppertino](https://github.com/codedgar/Puppertino) 的实现思路，**全部用 CSS 变量落地，禁止硬编码颜色，禁止 emoji 当图标（一律内联 SVG，`stroke="currentColor"`）**：

```css
:root {
  --font-stack: -apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Helvetica Neue", sans-serif;
  --accent: #0071e3;            /* Apple 蓝 */
  --accent-hover: #0077ed;
  --bg: #f5f5f7;                /* Apple 浅灰背景 */
  --surface: rgba(255,255,255,.72);  /* 毛玻璃卡片 */
  --text-1: #1d1d1f; --text-2: #6e6e73;
  --separator: rgba(0,0,0,.08);
  --radius-lg: 18px; --radius-md: 12px; --radius-pill: 980px;
  --shadow-card: 0 4px 24px rgba(0,0,0,.06);
}
```

- **毛玻璃**：侧边栏/顶栏/弹窗 `backdrop-filter: saturate(180%) blur(20px)`。
- **按钮**：主按钮 = accent 填充胶囊(`--radius-pill`)；次按钮 = 浅灰填充；文字按钮 = accent 无底。
- **状态徽章**：iOS 风 pill（浅色底+深色字：待审批橙、通过绿 #34c759、驳回红 #ff3b30、完成灰）。
- **表单控件**：输入框大圆角浅灰底聚焦蓝描边；开关=iOS switch；筛选切换=分段控件(segmented control)。
- **列表**：设置页风格分组卡片，行间 inset 分隔线；页面标题大字重 (34px/700) 仿 iOS Large Title。
- **动效**：150–250ms ease-out；弹窗 scale(0.96→1)+fade。
